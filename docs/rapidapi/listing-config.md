# RapidAPI Studio Listing Configuration

This document provides step-by-step instructions for configuring the **FX Intelligence API** listing in RapidAPI Studio.

## Prerequisites

- RapidAPI Provider account created at https://rapidapi.com/provider
- Cloud Run API service deployed and accessible at `https://api.fxintelligence.dev/v1`
- OpenAPI 3.1 spec available at `https://api.fxintelligence.dev/v1/openapi.json`
- `RAPIDAPI_PROXY_SECRET` stored in Google Cloud Secret Manager and mounted in the Cloud Run service
- Logo asset prepared (recommended: 256×256 PNG, transparent background)

## 2. Definitions Tab

### Upload Instructions

1. Navigate to the **Definitions** tab in RapidAPI Studio
2. Select **Import from URL**
3. Enter: `https://api.fxintelligence.dev/v1/openapi.json`
4. Click **Import**

RapidAPI will auto-generate:
- Interactive test interface for each endpoint
- Code samples in multiple languages
- Parameter documentation tables
- Response schema display

### Important Notes

- The OpenAPI spec uses `X-API-Key` as the security scheme for direct-access documentation
- **Do NOT** add `X-RapidAPI-*` headers to the spec — RapidAPI injects those automatically
- After import, verify that all four endpoints appear:
  - `GET /forecast/{asset}` — Probabilistic forecast with tradeability scoring
  - `GET /similarity/{asset}` — Historical similarity pattern matching (paginated)
  - `GET /state/{asset}` — Market regime and state detection
  - `GET /health` — Service health check

### Re-importing After Updates

When the OpenAPI spec is updated (new fields, parameters, or endpoints):
1. Go to **Definitions** tab
2. Click **Re-import** or delete existing definition and re-import from URL
3. Review the diff and confirm changes
4. Verify endpoint test interface still works

## 3. Gateway Tab

### Base URL Configuration

| Field | Value |
|-------|-------|
| **Base URL** | `https://api.fxintelligence.dev/v1` |

If using the raw Cloud Run URL instead of a custom domain:

| Field | Value |
|-------|-------|
| **Base URL** | `https://fip-api-<hash>-<region>.a.run.app/v1` |

### Settings

- **Protocol**: HTTPS (required)
- **Request timeout**: 30 seconds
- **Follow redirects**: No

### Transformations

No request or response transformations are needed. RapidAPI forwards requests as-is to the base URL with endpoint paths appended.

Example: A consumer calling `GET /forecast/EURUSD` on the marketplace results in RapidAPI forwarding to `https://api.fxintelligence.dev/v1/forecast/EURUSD`.

## 4. Security Tab

### Authentication Configuration

| Setting | Value |
|---------|-------|
| **Auth type** | No additional auth required |
| **Proxy Secret** | Configured automatically by RapidAPI |

### How It Works

RapidAPI handles all consumer-facing authentication. On every forwarded request, the RapidAPI proxy adds:

| Header | Purpose |
|--------|---------|
| `X-RapidAPI-Proxy-Secret` | Validates request originated from RapidAPI (matched against `RAPIDAPI_PROXY_SECRET` env var) |
| `X-RapidAPI-User` | Consumer's RapidAPI username |
| `X-RapidAPI-Subscription` | Consumer's plan name (`BASIC`, `PRO`, `ULTRA`, `MEGA`) |

Our API validates `X-RapidAPI-Proxy-Secret` server-side. No additional auth scheme needs to be configured in RapidAPI Studio because:
- RapidAPI authenticates consumers via their RapidAPI key
- Our proxy-secret validation confirms the request came through RapidAPI
- Internal tier mapping is handled by our auth middleware based on `X-RapidAPI-Subscription`

### Secret Setup

The proxy secret is generated by RapidAPI when you create the API listing. Store it in:
1. Google Cloud Secret Manager as `RAPIDAPI_PROXY_SECRET`
2. Cloud Run service configuration (mounted as env var)
3. `.env` file for local development (see `.env.example`)

## 5. Monetize Tab

### Plan Configuration

Configure four plans with the following settings:

| Plan | Price | Quota | Period | Rate Limit Type | Internal Tier |
|------|-------|-------|--------|----------------|---------------|
| **BASIC** | Free | 100 requests | Per day | Hard limit | RETAIL |
| **PRO** | $29/month | 5,000 requests | Per month | Hard limit | DEVELOPER |
| **ULTRA** | $79/month | 25,000 requests | Per month | Hard limit | RESEARCH |
| **MEGA** | $149/month | 100,000 requests | Per month | Hard limit | RESEARCH |

### Plan Details

#### BASIC (Free)

- **Object access**: All endpoints accessible, but response fields limited to RETAIL tier
- **Rate limit**: 100 requests/day (RapidAPI enforced)
- **Overage**: Blocked (429 returned by RapidAPI)
- **Endpoints available**: `/forecast/{asset}` only
- **Response fields**: 6 core fields (direction_probabilities, expected_move_pips, confidence_final, tradeability_score, tradeability_label, forecast_valid_until)

#### PRO ($29/month)

- **Object access**: DEVELOPER tier fields
- **Rate limit**: 5,000 requests/month (RapidAPI enforced)
- **Overage**: Blocked (429 returned by RapidAPI)
- **Endpoints available**: `/forecast/{asset}`, `/state/{asset}`, `/similarity/{asset}`
- **Response fields**: RETAIL fields + state_layers, layer_breakdown, similarity_matches, match_explanation, contributing_factors, execution_metrics

#### ULTRA ($79/month)

- **Object access**: RESEARCH tier fields
- **Rate limit**: 25,000 requests/month (RapidAPI enforced)
- **Overage**: Blocked (429 returned by RapidAPI)
- **Endpoints available**: All public endpoints
- **Response fields**: All fields except internal debug data

#### MEGA ($149/month)

- **Object access**: RESEARCH tier fields (same as ULTRA)
- **Rate limit**: 100,000 requests/month (RapidAPI enforced)
- **Overage**: Blocked (429 returned by RapidAPI)
- **Endpoints available**: All public endpoints
- **Response fields**: All fields except internal debug data

### Subscription-to-Tier Mapping

RapidAPI sends the plan name in `X-RapidAPI-Subscription`. Our API maps it to internal tiers:

```
BASIC  → RETAIL     (forecast only, 6 fields)
PRO    → DEVELOPER  (+ state, similarity, analytics fields)
ULTRA  → RESEARCH   (all public fields)
MEGA   → RESEARCH   (all public fields, higher quota)
```

### Rate Limiting Note

RapidAPI enforces quotas at the proxy layer. Requests exceeding the plan quota receive a 429 from RapidAPI **before** reaching our API. Our internal rate limiter is bypassed for marketplace requests (`req.isMarketplaceRequest === true`).

### Bandwidth

Each RapidAPI subscription includes 10 GB bandwidth per billing cycle. Typical forecast responses are < 5 KB, so bandwidth is unlikely to be a concern unless consumers poll aggressively.
